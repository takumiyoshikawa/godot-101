# ゲームルール

トランプ1組（52枚）+ Joker 1枚 = **53枚** を使ったマルチプレイヤー型のカード捨札ゲーム。

## 1. プレイヤー / 構成

- プレイヤー人数: **3〜5人**
- 使用デッキ: 標準トランプ52枚 + Joker 1枚
- ランクとカード番号: A=1, 2〜10=数字通り, J=11, Q=12, K=13, Joker=特殊
- スート: ♥/♦ = **赤**, ♠/♣ = **黒**, Joker = 色なし

## 2. ゲームの準備（セットアップ）

1. 山札（53枚）をシャッフルする。
2. 各プレイヤーに **手札を3枚ずつ** 配る。
3. 開始プレイヤーをランダムに決定する。
4. 捨て札は空の状態でスタート。捨て札はゲーム開始から終了まで全プレイヤー公開。

## 3. ターンの進行

各プレイヤーのターンは以下の順で進行する:

1. **ドローフェイズ**: 山札から 1枚引いて手札に加える（山札が空なら何もしない）。
2. **ディスカードフェイズ**: 手札から **必ず1枚** を選び捨て札の山の一番上に置く。
3. **効果解決フェイズ**: 直前に捨て札に置かれたカードに効果がある場合、その効果は **必ず** 発動する（任意発動と明記された効果を除く）。
4. ターン終了処理（後述「勝敗」を参照）→ 次のプレイヤーへ移行。

ターン順は座席順の時計回りで固定（脱落者はスキップ）。

### 3.1 効果発動のタイミング（最重要原則）

**カード効果は原則として「自身のターンの効果発動フェーズ（= 自分が能動的に1枚を捨てた直後）」のみで発動する。**

つまり:
- ✅ 自分のターンに自発的に捨てたカード → 効果発動
- ❌ 相手の攻撃で強制捨てされたカード → 効果は発動しない（ただ捨て札に積まれる）
- ❌ Joker による強制捨て → 効果は発動しない
- ❌ カード4 の不一致で公開され捨て札に置かれたカード → 効果は発動しない
- ❌ 防御や強化のために自分から公開して捨てたカード → そのカード自身の本来の on-discard 効果は発動しない

これにより**捨て札の連鎖は基本的に発生しない**。各ターンは「ドロー → 能動的に1枚捨てる → その1枚の効果が解決される（防御リアクションを含む） → ターン終了」というシンプルな流れ。

### 3.2 例外となる4種類のカード

| カード | 例外の種類 | 発動条件 |
|--------|-----------|---------|
| 5（防御） | 反応 | 自分が攻撃カードの対象になったとき、手札から公開して攻撃ベクトルを反転 |
| 6（防御） | 反応 | 自分が攻撃カードの対象になったとき、手札から公開して攻撃を無効化 |
| 9（特殊・受動） | 受動連鎖 | このカードが攻撃によって捨てられた直後、自分の手札が0枚になっていたら山札から1枚ドロー |
| 11（特殊・強化） | 強化使用 | 自分が攻撃カードの効果を発動するとき、追加でこのカードを捨てると攻撃効果が +1 される |

これら4枚は「捨て札になった瞬間に自動発動」ではなく、能動的な「使用宣言」または受動的な「特定状況のトリガー」によって発動する。

### 3.3 攻撃カードの解決時の防御機会

攻撃カード（1, 2, 3, 7, 10）の効果を発動する側が対象を指定したとき、対象は防御カード（5/6）を手札から公開する機会を持つ。詳細は §6.5 参照。

## 4. 勝敗条件

- **敗北（脱落）**: 任意のプレイヤーのターンが終了したタイミングで、**手札が0枚** のプレイヤーは脱落する（自分のターンか他人のターンかを問わない）。複数人が同時に脱落する場合もある。
- **勝利**: **山札が0枚になったターンの終了処理** で、まず脱落判定（手札0の脱落）を行い、その後 **生き残ったプレイヤーのうち手札枚数が最少の者** が勝者。
- **同枚数で並んだ場合のタイブレーク**: そのターンの終了プレイヤーから時計回りに最初に該当した1人を勝者とする。

> ターンの最中に手札が0になっただけでは脱落しない。例えば攻撃で手札を全部取られても、その後カード9でドローして手札を回復できれば生存できる。

## 5. カード効果一覧

凡例: 種別＝（攻撃 / 防御 / 特殊 / 効果なし）, 任意 = プレイヤー判断で発動可否を選べる, 必須 = 必ず発動。

> **発動タイミングの再掲**: 下記「効果」欄に書かれた処理は §3.1 の通り **自身のターンの効果発動フェーズ** でのみ発動する。例外は §3.2 の防御 5/6（反応）、特殊 9（受動）、特殊 11（強化）の4種のみ。それ以外のカードは強制捨てや Joker、カード4 の不一致で捨て札に置かれても効果を発動しない。

| ランク | 種別 | 任意/必須 | 効果 |
|------|----|---------|----|
| 1 (A) | 攻撃 | 必須 | 自分以外のプレイヤーを1人選ぶ。対象は **手札を裏向きに並べ**、攻撃者は位置のみを指定する。指定された1枚が対象の捨て札になる。 |
| 2 | 攻撃 | 必須 | 自分以外のプレイヤーを1人選ぶ。対象は **手札を裏向きに並べ**、攻撃者は位置を **2つ** 指定する。指定された2枚が捨て札になる。対象の手札が1枚なら1枚のみ捨てる。 |
| 3 | 攻撃 | 必須 | 自分以外の **全プレイヤー** がそれぞれ自分の手札から1枚 **自分で選んで** 捨てる（カード3は対象が中身を見て選べる例外）。 |
| 4 | 特殊 | 必須 | 赤/黒のいずれかを宣言。山札の最上段を公開する。色が一致 → 自分の手札に加え **再度4の効果を続ける**。一致しない → 公開したカードを捨て札に加える（その効果は通常通り発動）。Joker は不一致扱い。山札が0なら効果終了。 |
| 5 | 防御 | 任意 | 自分が攻撃カードの対象に取られたとき、手札から公開してよい。**攻撃のベクトルを反転** する（現在の「攻撃元 → 自分」を「自分 → 攻撃元」に入れ替える）。新しい対象は再びカード5を公開してさらに反転できる（A→B→A→B→… と何度でも往復可）。 |
| 6 | 防御 | 任意 | 自分が攻撃カードの対象に取られたとき、手札から公開してよい。**その攻撃を無効化する**（自分への影響なし）。 |
| 7 | 攻撃 | 必須 | 自分以外のプレイヤーを1人選ぶ。対象は手札を **裏向きに並べ**、攻撃者は位置（左から N 枚目など）だけを指定する。指定された1枚が攻撃者の手札に加わる（捨て札にはならないので効果は発動しない）。対象の手札が0なら何も起きない。 |
| 8 | 特殊 | 必須 | 山札から1枚ドローし手札に加える。山札が0なら何もしない。 |
| 9 | 特殊 | 必須 | このカードが **攻撃カードの効果によって** 捨て札に置かれた直後、そのプレイヤーの手札が **0枚になっていた場合**、山札から1枚ドローして手札に加える。山札が0なら何もしない。通常のターン中の自発捨て札では発動しない。 |
| 10 | 攻撃 | 必須 | カード1と同じ効果（対象が裏向きに手札を並べ、攻撃者が位置を指定して1枚捨てさせる）。 |
| 11 | 特殊 | 任意 | **攻撃カードの効果発動の直前**、その攻撃を行うプレイヤー自身が手札から追加でこのカードを捨て札に置いてよい。そうした場合、攻撃の効果値を **+1** する（例: 1/10は捨てさせる枚数が2になる、2は3になる、3は全員が2枚捨てる、7は奪う枚数が2になる）。複数枚の11を同時に重ねても良い（累積）。 |
| 12 | 効果なし | — | 捨てても何も起きない。 |
| 13 | 効果なし | — | 捨てても何も起きない。 |
| Joker | 特殊 | 必須 | プレイヤーを **2人選ぶ**（順序付き: 「最初」「最後」）。最初のプレイヤーの手札枚数を、最後のプレイヤーの手札枚数と一致するまで調整する。<br>・最初 > 最後 → 最初のプレイヤーが手札から差分枚数を選んで捨てる<br>・最初 < 最後 → 最初のプレイヤーが山札から差分枚数をドローする（山札が尽きたらそこで停止） |

## 6. 効果解決の細目（実装上の確定事項）

### 6.1 攻撃カードの解決手順（一般形）

1. 攻撃の発動者が必要なパラメータ（対象、色など）を決める。
2. **カード11による強化** をここで処理する（攻撃者が任意で11を捨てて効果値+1。複数枚累積可）。
3. 対象プレイヤーが順に防御カード（5/6）を公開する機会を得る。
   - 複数対象の攻撃（カード3）の場合、解決順は攻撃者から時計回り。各対象が独立に防御の機会を持つ。
4. 防御が成立したら攻撃の効果はその対象には適用されない。リダイレクト（5）の場合は新対象に対して 3〜4 を繰り返す。
5. 防御がなかった対象に対し、攻撃効果を適用する。
6. 強制捨てされたカードは捨て札に積まれるが、§3.1 の通り **on-discard 効果は発動しない**。例外はカード9（受動：手札0で1ドロー）のみ。

### 6.2 強制ディスカード時のカード選択

- カード1, 2, 7, 10 は **裏向き位置指定方式**: 対象は手札を裏向きに並べ、攻撃者は左から N 枚目という位置のみを指定する。中身は確認できない。
  - 並べる順番は対象の自由（実装上はゲーム内の手札順を保ったまま裏向き表示する）。
- カード3 は対象自身が **中身を見て** 1枚選んで捨てる例外。
- 攻撃で対象の手札が **強制ディスカード枚数より少ない場合**、出せるだけ出す（足りない分は無視）。

### 6.3 連鎖の有無

- 強制捨て・Joker 強制捨て・カード4 の不一致捨て札ではいずれも on-discard 効果が発動しないため、**連鎖は発生しない**。各ターンで実質的に発動する効果は「自発捨て1枚 + その派生（防御リアクションやカード11の強化）」のみ。
- カード9 のみ受動的に「攻撃で捨てられて手札0」で1ドローする例外がある。これは厳密には「効果連鎖」ではなく独立した受動トリガー。

### 6.4 山札枯渇時のハンドリング

- ドロー要求（ターン開始ドロー、カード4/8/9、Joker等）で山札が0なら、その分のドローは **失敗（何もしない）** とし、ゲームは続行。
- 山札が0になった瞬間ではなく、**「山札が0になったターンの終了時点」** に勝利判定を行う。

### 6.5 防御カードの相互作用（カード5の方向反転モデル）

- 攻撃の状態は「攻撃元 → 対象」というベクトル。カード5を対象が公開すると、このベクトルが **反転して「対象 → 攻撃元」** になり、攻撃元が新しい対象になる。
- 反転後の新しい対象も同様にカード5を公開できる。再反転すると「攻撃元 → 対象」に戻る。理論上は何度でも往復する（カード5の手持ちが続く限り）。
- カード6（無効化）はベクトルそのものを消す。攻撃の効果は完全に発生しない。
- 防御カード（5/6）を公開したことで、その防御カードは捨て札に置かれる。ただし防御カード自身の効果は **すでに発揮済み（反転 / 無効化）** で、再度発動することはない。
- カード3（全体攻撃）の場合、各対象は独立に防御の機会を持つ。カード5を公開した対象だけ、攻撃者へとベクトルが向かう（攻撃者は1枚追加で捨てさせられる）。

### 6.6 Joker 解決順

- Joker を **自身のターンで捨てた** プレイヤーが2人を選び、解決する（同一プレイヤーを最初・最後に指定することも可能だが、その場合は何も起きない）。
- Joker 自体も §3.1 の通り「自発捨て」でしか効果を発動しない。攻撃で強制捨てされた Joker やカード4 でめくられた Joker は効果を発動せず、ただ捨て札に積まれる。
- 「最初のプレイヤー」が **捨てる場合**、捨てるカードは最初のプレイヤー自身が選ぶ。Joker 強制捨ても §3.1 通り on-discard 効果を発動しない。
- 「最初のプレイヤー」が **ドローする場合**、山札の上から順に必要枚数を引く。山札が途中で尽きたらそこで停止。

## 7. プレイヤー脱落の取り扱い

- ターン終了時に手札0枚になったプレイヤーは、その時点で **手札・席を解除** する（持っていたカードは無いので何も起きない）。
- 攻撃の対象選択肢から脱落者は除外される。
- ターン順から脱落者はスキップされる。
- 脱落者しか残らない（生存者≦1人）にはなり得ない（手札が0で脱落するため、必ず生存者は手札を保持している）。

## 8. 決定事項ログ

実装にあたり確認した曖昧点と、その確定回答を記録する:

| # | 論点 | 決定 |
|---|------|------|
| 1 | 山札0枚時に手札最少で並んだ場合の勝者 | **そのターンの終了プレイヤーから時計回り** で最初に該当する1人を勝者とする |
| 2 | カード5のリダイレクト先 | 攻撃の方向を **ベクトルとして反転**。A→B を B→A、再度公開で A→B に戻る。何度でも往復可 |
| 3 | カード7（および1, 2, 10）の選択方法 | 対象が **手札を裏向きに並べ**、攻撃者が **位置のみ** を指定 |
| 4 | カード11の累積上限 | **上限なし**（攻撃者の手札にある11の枚数だけ重ねられる） |
| 5 | カード4の連続ドロー停止可否 | ルール文通り **強制継続**（不一致が出るか山札が尽きるまで止められない） |

| 7 | ターン終了時の手札0判定範囲 | **全プレイヤー** をそのターン終了時に一斉判定 |

| 9 | カード効果の発動条件（全カード共通） | **自身のターンの効果発動フェーズ（NORMAL_TURN）のみ**。例外は防御 5/6（反応）、特殊 9（受動：攻撃で捨てられて手札0）、特殊 11（強化使用）の4種だけ。Joker やカード4も自発捨てのみで発動 |
