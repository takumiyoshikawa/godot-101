# 2026-04-27 godot-101 ブートストラップ

Godot 4.6 + ヘッドレス AI コーディングワークフローの土台を一通り構築し、
パイプラインがブラウザ表示まで通ることを確認した日。

参考にしたもの:

  Godot + AIエージェントによる「コード編集 → headless export → ブラウザ確認」ループ
- 参考リポ: <https://github.com/abagames/headless-godot-skill-kit>
  上記ループを再利用可能な skill kit として整理したもの

## 確定した方針

- **構成**: feature-based ディレクトリ (`features/`, `assets/`, `autoload/`, `addons/`)
  + ルートに `main.tscn`/`main.gd` のエントリスタブ。
- **シーン編集**: `.tscn` を**生テキスト編集しない**。`tools/godot_apply_patch.gd`
  に JSON を渡して適用する片方向フロー。
- **テスト**: GUT は採用せず、kit 流の素朴な GDScript ランナー
  (`tools/tests/run_tests.gd`)。`main.gd` の test hooks
  (`enable_test_mode` / `force_reset_for_test` / `step_for_test` / `get_metrics`)
  を契約として固定。
- **エクスポート**: Web のみ。`build/web/index.html` 出力。`web/custom_shell.html`
  で 960×540 のレターボックス + ローダーUI。
- **規約の置き場**: `.agents/skills/headless-godot/` (kit 由来の SKILL.md と
  サブスキル) を権威ソースとし、プロジェクト固有の運用ルールを `CLAUDE.md` に。

## やったこと

### 1. プロジェクト雛形

- `brew` 経由で Godot 4.6.2 を導入 (`/opt/homebrew/bin/godot`)。
- 最小構成の `project.godot` を手書き → `godot --headless --editor --quit` で
  `.godot/` キャッシュ生成を確認。
- 当初は `scenes/main.tscn` の構造で作ったが、kit のフラット構成に合わせて
  ルート直下の `main.tscn`/`main.gd` に再配置。

### 2. headless-godot-skill-kit の取り込み

`/tmp/hgsk/headless-godot-skill-kit` に clone して中身を読み、以下をコピー:

| コピー元 | コピー先 |
|---|---|
| `.agents/skills/headless-godot/SKILL.md` | 同じ |
| `.agents/skills/headless-godot/skills/*.md` (4枚) | 同じ |
| `.agents/skills/headless-godot/tools/godot_apply_patch.gd` | `tools/godot_apply_patch.gd` |
| `templates/godot-base/main.{gd,tscn}` | ルート |
| `templates/godot-base/audio_synth.gd` | ルート |
| `templates/godot-base/metrics_tracker.gd` | ルート |
| `templates/godot-base/tools/create_main_scene.gd` | `tools/` |
| `templates/godot-base/tools/tests/run_tests.gd` | `tools/tests/` |
| `templates/godot-base/web/custom_shell.html` | `web/` |

### 3. 自前で書いたもの

- `export_presets.cfg` — Web プリセット。`export_path` を
  `res://build/web/index.html` に変更、`exclude_filter` をうちのレイアウトに合わせ調整。
- `.gitignore` — `.godot/`、`.tmp-godot-*/`、`logs/`、`build/*` (ただし
  `build/web/` は残す) を除外。
- `CLAUDE.md` — エージェント向け運用ルール。`.agents/skills/headless-godot/SKILL.md`
  を権威とし、ワークフロー4段階(patch→smoke→tests→export)とローカル確認手順を記載。
- `tools/rebuild_web.sh` — smoke→tests→export を順に流すワンショットスクリプト。
- `worklog/` — 本ファイルのディレクトリ。

### 4. Web エクスポートテンプレート

- 初回 `--export-release "Web"` で
  `~/Library/Application Support/Godot/export_templates/4.6.2.stable/web_nothreads_*.zip`
  が無くて失敗。
- GUI (`godot -e`) を起動 → エディター → エクスポートテンプレートの管理 →
  ダウンロードしてインストール、で導入。
- 以降ヘッドレス export が成功するように。

### 5. パイプライン全段検証

- smoke run: エラー無し
- `tools/tests/run_tests.gd`: `tests passed` (`logs/test.json` に metrics 出力)
- `--export-release "Web" build/web/index.html`: `build/web/` に
  `index.html` / `index.js` / `index.wasm` (~36MB) / `index.pck` 等を生成
- `python3 -m http.server --directory build/web 8000` → ブラウザで
  ローダー(0→100%) と空キャンバス表示まで確認 (中身が空の Node2D
  なので真っ黒キャンバスが正常)

## 詰まったポイント

1. **export 先のディレクトリが事前に存在しないと失敗する**
   `エクスポート: 対象となるフォルダが存在しないか、アクセスできません: "build/web"`。
   `mkdir -p build/web` を `rebuild_web.sh` に組み込んで対処。
2. **Web export は `file://` で動かない**
   ローカル動作確認には `python3 -m http.server` などの HTTP サーバが必須。
3. **ヘッドレス export 中に GUI エディタを開いたままだと衝突しがち**
   テンプレートDL後は GUI を閉じてから headless コマンドを叩く運用にした。

## 残タスク

元プランの項目4「デバッグ描画ヘルパー」(衝突判定矩形を可視化してスクショ→
エージェントへの視覚フィードバック) は未着手。実ゲームで衝突周りを書き始めた
タイミングで作るのが自然。

ゲーム本体 (`main.gd` のスタブ置き換え、`features/` 配下の追加) はまだ無し。

## 動かし方 (次回以降のメモ)

```bash
# フルパイプライン
tools/rebuild_web.sh

# ブラウザ確認
python3 -m http.server --directory build/web 8000
# → http://localhost:8000/

# 個別実行
godot --headless --path . --quit-after 60                                          # smoke
godot --headless --path . --script res://tools/tests/run_tests.gd                  # tests
godot --headless --path . --export-release "Web" build/web/index.html              # export

# シーン編集 (生テキスト編集禁止)
godot --headless --path . --script res://tools/godot_apply_patch.gd -- <patch.json> --dry-run
godot --headless --path . --script res://tools/godot_apply_patch.gd -- <patch.json>
```
